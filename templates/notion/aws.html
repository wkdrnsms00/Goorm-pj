AWS

AWS(Amazone Web Service)

온라인 쇼핑몰 AMAZONE에서 개발한 클라우드 컴퓨팅 플랫폿으로 컴퓨팅 서비스, 네트워킹 서비스, 스토리지 서비스,데이터베이스 서비스 등 다양한 서비스를 제공하고 있다.


리전(Region)

물리적으로 거리가 다른 지역과 같은 서버를 사용하게 되면 속도 문제가 발생할 수 있다. 그리고 재해와 같은 문제 발생시 대체 서버가 없으면 서버가 정지된다. 아마존은 이러한 상황이 발생하지 않게 하기 위해 전세계 주요 국가에 여러 서버를 구축하는데 이를 리전이라 한다. 사용자는 해당 위치에서 가장 가까운 리전을 클라우드 서비스를 사용하면 된다.

주의할점은 리전마다 제공되는 서비스가 다르므로 내가 사용할 서비스가 선택한 리전에서 제공하는지 확인해야 된다.


가용영역(Avilability Zone)

가용영역은 데이터 센터라고 생각하면 된다. 하나의 리전은 여러 가용영역을 보유하고 있으며 가용영역은 물리적으로 떨어져있다. 재해, 정전과 같은 상황이 발생하여 하나의 가용영역을 사용할 수 없을 때 다른 가용영역을 사용해서 서비스를 정상적으로 진행할 수 있다.

S3

스토리지

데이터를 저장하는 저장소의 역할을 수행하는 요소여러개의 스토리지를 하드웨어 고장, 하드디스크 손상을 대비하기 위한 데이터 백업용으로 사용할 수 있음.(스냅샷 기능도 사용 가능)

DAS(Direct Attached Storage)

직접 서버에 연결하는 저장소

NAS(Network Attached Storage)

스토리지를 네트워크로 연결하는 파일 기반 저장소

SAN(Storage Area Network)

별도의 스토리지용 네트워크를 구성(Fibre Channel)하는 블록 기반 저장소

S3(Simple Storage Service)

확장성이 뛰어나 거의 무한대로 저장 가능한 스토리지로 사용하는 만큼 비용을 지불해야 된다.파일 형태로 스토리지 공간에 저장하며 저장 공간을 버킷(Bucket), 저장하는 각각의 파일을 오브젝트(객체)라 한다.

버킷 설정에 따라 오브젝트를 공개할 수 있고, 비공개 할 수 있다.

버킷 생성 및 설정

Public

모든 사람이 오브젝트를 접근할 수 있게 설정하기 위해 ACL활성화 선택, 모든 퍼블릭 엑세스 차단 체크 해제 진행

파일 업로드 및 설정

원하는 파일 업로드 후 아래 권한 부분에서 퍼블릭 읽기 엑세스 권한 부여 선택

확인

업로드한 파일 선택 후 객체 URL 확인

private의 경우 ACL 비활성화 선택, 모든 퍼블릭 엑세스 차단 체크로 생성하면 된다. 단, private 버킷에서 public으로 변경은 안된다.파일의 입출력이 적고 단순 데이터 백업용도로 사용할 때는 Glascier를 사용하는게 S3를 사용하는 것 보다 저렴하다.

Auto Scaling

Auto Scaling

서버 구성 시 서비스가 지속적으로 제공될 수 있는 것이 중요하다. Auto Scaling은 서버에 부하가 올 경우 다운되는 것 을 방지하기 위해 인스턴스를 자동으로 늘려주는 서비스이다.

과정

시작 구성 생성

이미지, 보안 그룹, 키페어 등.. 필요한 정보 작성

Auto Scaling 생성

1단계 ~ 7단계를 거쳐 생성

1단계 - 시작 템플릿 또는 구성 선택

2단계 - 인스턴스 시작 옵션 선택

VPC, 서브넷 설정

3단계 - 고급 옵션 구성

로드밸런스 연결 및 모니터링 설정

4단계 - 그룹 크기 및 크기 조정 정책 구성

Auto Scaling 정책 설정

5 ~ 6 단계는 임의로 설정해주면 된다.

7단계 - 검토

지금 까지 설정한 내용을 다시 확인하는 부분이다.
