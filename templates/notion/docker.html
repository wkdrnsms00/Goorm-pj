DOCKER

Docker 정리

  • 서버
    • 서비스를 제공하는 시스템 (컴퓨터)
  • 인프라
    • 서버를 구성하는 환경
    • 서버라는 시스템을 포함해서 네트워크 / 스토리지

  • 구성 방식
    • 실제 시스템들을 이용하는 형태 (온프레미스)
    • 서버를 구성할 때
      • 실제 운영체제에 직접 서비스를 구성
      • 가상머신을 이용하는 형태
      • 컨테이너를 이용하는 형태

    • 가상 머신 이용 시
      • 서버 가상화
      • 클라우드
        • 퍼블릭
        • 프라이빗(오픈스택)
    • 컨테이너
      • 쿠버네티스

  • 가상화 개념
    • 물리적인 장치에 대해 논리적인 개념으로 구성
    • 논리적으로 여러 장치를 하나로 통합 혹은 하나의 장치를 여러 개로 분할 하는 기술

  • 종류
    • 시스템 가상화
    • 네트워크 가상화 - vlan, VPN (SDN, NFV)
    • 스토리지 가상화 - LVM, S3(오브젝트 스토리지) → SDS 등

    • 시스템 가상화

    1) 가상머신

    → 하이퍼바이저로 관리하는 가상머신

    가상의 운영체제가 포함

    리소스낭비 / 오버헤드가 증가

    2) 컨테이너

    → 컨테이너런타임(엔진)으로 관리하는 컨테이너

    어플리케이션만 격리

    리소스 낭비 감소 / 오버헤드도 감소

    배포 속도가 빠름


도커(DOCKER)

  • 리눅스 컨테이너를 관리하는 컨테이너 런타임(엔진)중의 하나
  • 컨테이너 런타임은 컨테이너를 생성/관리 역할
  • 리소스들에 대한 격리 ( 프로세스 / 파일시스템)

이미지

⇒ 어플리케이션 실행에 있어서 필요한 파일들의 집합

컨테이너

⇒ 이미지를 실행한 상태

실행파일 : 프로세스 = 이미지 : 컨테이너

  • 기능
    1. 이미지 생성
    1. 이미지 공유
    1. 컨테이너 실행

  • 오브젝트
    1. 이미지
    1. 컨테이너
    1. 네트워크
    1. 볼륨

  • 기반 기술 (원래 있는거)
    • namespace (이름공간) : 각각의 리소스를 격리
    • cgroup (컨트롤그룹) : 리소스 관리를 위한 그룹
    • 가상브릿지/vNIC : 컨테이너에서 사용할 네트워크 환경을 제공하기 위한 기능
    • 파일시스템 : 컨테이너의 데이터 관리를 위해

  • 도커 사용 시 사용자
    1. root 사용자
    1. sudo 명령어 사용 가능한 사용자
    1. docker 그룹의 구성원

  • 도커 이미지
    1. 이미지 이름

      이미지명 : 태그

      → 이미지명은 일반적으로 어플리케이션 종류 태그는 버전인 경우가 다수

    1. 명령어
    • docker image pull : 사용할 이미지를 다운로드 ( 기본적으로는 docker hub 에서)
    • docker image ls : 다운로드한 이미지 목록 확인
    • docker image rm : 로컬 이미지를 삭제
    • docker image inspect : 로컬 이미지의 상세정보 확인

    ** 참고

    이미지의 태그는 생략하면 무조건 latest 로 지정 가급적이면 특정 태그를 지정해주는 것이 좋음

    latest 는 사용하지 않는 것을 권장

  • 컨테이너 관리 명령어
    • docker container create : 컨테이너 생성
    • docker container start : 생성한 컨테이너를 실행
    • docker container run : 생성 + 실행

    • create 중요 옵션

    -it : 입출력이 필요한 경우 (쉘을 실행할 때)

    —rm : 컨테이너 종료 시 자동 삭제

    —name : 컨테이너 이름 지정

    • start 옵션

    -ai : 입출력이 필요한 경우 사용 (create 에서 -it 필수)

    • run 옵션

    -it : 입출력이 필요한 경우 (자동연결)

    -d : 연결이 불필요한 경우 (백그라운드 실행)

    • 편의 옵션

    —rm

    —name

    …

    • docker container ls : 컨테이너 목록 확인 → docker ps 로 줄여서 사용, 사용 시 -a 옵션으로 중지 상태의 컨테이너까지 확인가능

  • 컨테이너 생성 및 실행 시 주의 사항
    • 이미지 종류에 따라 (실행하는 프로세스) 옵션이 다름
    • 쉘을 실행하는 이미지는 생성 시 필수적으로 -it 옵션 필요 → 입출력장치 연결

      실행 시 -ai 옵션을 사용할 경우만 연결

    • 서비스(데몬프로세스)를 실행하는 이미지는 생성 시 옵션 불필요

      실행 시 옵션(-ai) 사용하면 이후 작업 불가

    • run 명령어 사용 시에는 -it 옵션은 동일 → 연결이 필요없는 경우는 -d 옵션 사용 (데몬프로세스 종류들)
