LINUX

Linux 정리

  • Kernel : 핵심 Interface로 하드웨어와 사용자 사이에서 자원 처리
  • Shell : 사용자가 입력하는 명령어를 이해하고 처리하는 프로그래밍 계층
  • Directory: Home (사용자가 파일 생성 및 삭제) Work (명령어 실행)
  • 핵심 명령어 : Command [Option]..[Argument]..
    • uname -s (os 종류) -r (커널 버전)
    • grep -i (문자에 상관없이),-n (숫자와 함께), -v(을 제외하고), -c(넘버링과 함께), -w (단어마다) …. [파일명] : 파일 내에서 문자열 검색
    • cp A(를) B(로), cp A,B,C(들을) 폴더(로) 복사, rm -r 폴더 삭제, rm -rf 폴더 (서브를 포함하여 삭제)
    • locate 패턴, find 경로 찾기
  • LINK
    • Hard Link : 파일의 물리적 주소 연결, i-node 값과 파일에 대한 권한 공유, 원본과 같기 때문에 원본이 지워져도 이용 가능 . In 명령어
    • Symbolic Link: 파일과 폴더의 논리적 주소 연결, i-node 공유하지 않고 원본과 구분되기 때문에 원본이 지워지면 사용 불가. In -s 명령어
  • Vi Editor 핵심 명령어
    • Command Mode ( Default ): COPY (yy,yw), PASTE (p), DELETE (dd,dw), MOVE ( number G)
    • Edit mode : INSERT, REPLACE
    • Extended Mode (Setting, write and quit, search and exchange) : :wq (저장 후 나가기), : x 경로/파일명 (새로운 파일로 저장), :set number (넘버링)
  • Archive : 파일 압축 형식에 따라 네 가지 분류
    • TAR_FILE : $ tar cf,cvf
    • TAR_GZIP_FILE : $ tar zcf,zcvf
    • TAR_BZIP2_FILE : $ tar jcf,jcvf
    • TAR_XJ_FILE : $ tar Jcf,Jcvf
    • 각각 f 앞 알파벳을 t로 (파일 리스트), x로 (발행) 변경해주면 기능 변경
  • 처리
    • 프로그램이란 소스 코드로 만들어져 보조기억장치에 저장되는 이진 파일
    • 처리란 프로그램을 메모리에 적재하여 CPU에 의해 실행하는것
      • ps -ef | grep [Pattern]: 패턴을 포함하는 실행중인 프로세스 찾기
      • ps aux : 시스템 상에서 실행중인 프로세스 자세히 검색
      • kill -9 Pidnumber : 해당 넘버를 가진 프로세스 종료
      • pstree -p : 프로세스를 pid값을 가진 트리형태로 정렬
  • 스케쥴링 (작업 예약)
    • at : 한 번만
      • at -l, atq : 실행 대기중인 프로세스 리스트
      • at -d, atrm : 삭제
    • crontab: 주기적으로
      • crontab -e (생성 및 편집) -l (리스트) -r (삭제)
      • 크론탭에 의해 크론이 실행됨
        • 시스템 크론: /etc/crontab, /etc/cron.d
  • 파일과 사용자
    • 접근 허용
      • 사용자/그룹/그외에 에게 RWX 권한 부여 (2^2,2^1,2^0 으로 차례대로 4,2,1번 권한)
      • chmod [option] 폴더나파일 : chmod 444 A.txt
      • 사용자에 대한 기본 세팅은 /etc/profile, ~/.profile, ~/.bash_rc 에
      • 숫자가 4개인 권한은 특수권한
        • setuid: 파일 소유권자와 같은 권한으로, 4***, ex)chmod 4755 set.exe 하면 -rwsr-xr-x
        • setgtd: 파일 소유그룹과 같은 권한으로, 6***
        • sticky bit: 누구나 파일 및 폴더를 만들 수 있으나 삭제는 소유자나 루트만 가능, 1***
    • DISK
      • Serctor > Track > Cylinder > Partition > Disk
      • 파티션 타입 : MBR, GPT, df -Th 명령어로 조회
      • 인터페이스 (디스크에 연결) 타입 : E-IDE,SATA,SCSI,SA-SCSI(SAS)
      • 파일 시스템 (디스크 관리) : FAT32, NTFS (Windows), ext4 (Linux), xfs (Redhat), ufs (Unix), zfs (Solaris)
      • Disk 파티션 과정
        • 설치 및 인식
        • 파티셔닝
          • fdisk -l : 디스크 정보 확인 후 MBR에 파티셔닝할 장치 선택
          • fdisk /dev/sdb
        • File System 설치
          • mkfs -t ext4 /dev/sdb1, mkfs -t xfs /dev/sdb2
        • mount
          • mkdir /test1,2,3
          • mount /dev/sdb1 /test1, mount /dev/sdb2/test2
          • /etc/fstab (부팅시 자동 마운트)
      • Disk Logical Volume 사용 과정
        • LVM : 디스크 크기에 구애 받지 않도록 다수의 디스크를 유연하게 사용
        • 파티셔닝 > Physical Volume으로 전환 > PVs 를 볼륨 그룹에 바인드 > LV로 전환
          • fdisk /dev/sdb s
          • pvcreate /dev/sdb
          • vgcreate grp1 /dev/sdb1
          • lvcreate -l 248 grpl -n mylvm1
          • mkfs.ext4 /dev/grp1/mylvm1
          • mkdir /mnt/lvm, mount /dev/grp1/mylvm1 /mnt/lvm
  • Systemctl
    • init 명령어를 대체하기 위한 Systemd 명령어
    • 명령어 구조 systemctl [option][command][unitname]
      • command : status, start, restart enable, stop, disable, mak, umask, reload, isolate
  • Log
    • rsyslogd (/var/log, syslog)에 의한 로그 관리와 systemd-journald (/run/log/journal, journal data)에 의한 로그 관리로 나뉨
      • rsyslogd
        • /etc/rsyslog.co에 로그 유형과 우선순위 설정 되어 있음
        • 우선순위 0(높음)~7(낮음)
        • 조치: authpriv (보안,인증), omusrmag (사용자 지정)
      • systemd-journald
        • journalctl -r reads only (이진파일리고 휘발성)
        • 영구파일로 저장하려면
          • mkdir /var/log/journal
          • chown root:systemd-journal
          • systemctl reater systemd-journald
    • logger
      • /var/log/messages에 저장됨
      • logger -p (priority) cron.info “cronlog1”

    • nmcli 명령어

  • Firewall
    • 핵심 명령어 firewall-cmd -pernamenet -add
  • Storage
    • DAS (Direct-Attached Storage)
      • 로컬에서 데이터 공유, 설치가 쉬움, 작은 조직에 적합, 하드 드라이브 저장소, 실용적이나 공간이 제한적
    • NAS (Network-Attached Storage)
      • 로컬에서 데이터 공유, 다수의 파일 서버를 결합, 공간 절약 하나 확장성과 속도에 제한이 있음 (LAN 트래픽 상승시)
    • SAN (Storage Area Networks)
      • 로컬 밖에서 파일 공유 가능
      • IP SAN : 블록 단위 데이터를 전송하기 위해 iSCSI 프로토콜을 이용 (TCP/IP) 적은 비용, 단순함 ,
        • iSCSI (Internet Small Computer Systems Interface)
          • The protocol allows clients (called initiators) to send SCSI commands (CDBs) to storage devices (targets) on remote server.
          • iSCSI Qualified Name (IQN)
            • iSCSI uses TCP for the protocols itself, with higher-level names used to address the objects within the protocol IQN
              • iqn.Date.reversed Domain:sub Domain

  • DNS
    • 사용자가 도메인을 입력하면, DNS가 IP 주소를 회신하여 접속이 가능해짐
      • Caching-only DNS Server
        • yum -y install bind bind-chroot
        • vi /etc/named.conf
          • listen-on port 53 any, listen-on-v6 port 53 none, allow-query any
        • systemctl start named, systemctl enable named
        • firewall-cmd –permanent –add-service=dns
        • firewall-cmd –reload
        • firewall-cmd –permanent –add-port=53/tcp, firewall-cmd –permanent –add-port=53/udp, firewall-cmd –reload

      • Name Server
        • yum install -y httpd
        • systemctl start httpd
        • systemctl enable httpd
        • firewall-cmd –permanent –add-service=http
        • firewall-cmd –reload
        • vi /etc/named.conf (Service Area for BIND)
          • zone “linux.com” IN {
            • type master;
            • file “linux.com.db”;
            • allow-update {none;}; (non Second slave)};
        • named-checkconf
        • cd /var/named
        • cp named.localhost linux.com.db
  • Network File System (NFS)
    • 분산 파일 시스템으로, 사용자가 로컬 저장소 사용하듯 서버의 저장소를 사용
      • Commands
        • yum -y install nfs-utils
        • vi /etc/exports
          • [Directorytoshare][Objective](Option)
          • /share IP(ro,syn,sec=sys)
        • mkdir /share, chmod 777 /share, touch /share/1234, ls /share
        • systemctl restart nfs-server, systemctl enable nfs-server
        • exportfs -v (Confirming)
        • firewall-cmd –permanent –add-service mountd, firewall-cmd –permanent –add-service rpc-bind (Check ShareDirectory), firewall-cmd –permanent –add-service nfs, firewall-cmd –reload
        • Clients
          • rpm -qa nfs-utils (Display installed package, showmount -e 10.0.2.15 (Confirming Directory on 10.0.2.15/export)
          • mkdir /nfs_share
          • mount -t nfs 10.0.2.15:/share /nfs_share
          • cd /nfs_share/
